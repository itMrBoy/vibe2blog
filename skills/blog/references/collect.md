# 素材采集

目标:为一篇**聚焦单一主题**的随笔,凑齐够用的事实素材。不求全,只取与主题相关的。

## 先定题,再采集

动手前先和作者敲定:这篇就讲什么、边界在哪。一篇只讲一件事。题太宽就先切小。

## 四个素材源(让作者勾选,可补充)

- **当前对话 / 会话过程**:这次到底做了什么、卡在哪、怎么权衡、试错了几轮。最鲜活的素材,往往是博客的主线。从对话里摘:决策点、放弃的方案、踩的坑、最后的取舍。
- **git 提交与代码变更**:`git log --oneline` 看脉络,`git diff` / `git show` 看具体改动。用来核对事实,避免凭印象写错。引用具体文件/函数时以这里为准。
- **llmdoc / 计划文档**:项目已有的沉淀(如 `llmdoc/`、设计/计划文档)。用来补背景、对齐术语,别和已记录的结论打架。
- **作者口述要点**:作者临时给的 bullet 或主题方向。优先级最高——他点了方向,就顺着写。

## 采集原则

- 只收主题相关的。跑题的素材再有意思也先放下。
- 事实(数字、文件名、报错)以 git / 实际输出为准,不编。
- 拿不准的地方标出来,成稿时问作者,别糊弄过去。
